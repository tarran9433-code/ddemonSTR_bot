# 💾 Настройка Railway Volume для хранения данных

## 🎯 Зачем нужен Volume

Railway - это эфемерная среда. При каждом перезапуске:
- Файлы в контейнере удаляются
- `data/users.json` теряется
- Настройки пользователей сбрасываются

**Volume решает эту проблему** - данные сохраняются между перезапусками.

---

## 🛠️ Создание Volume в Railway

### Шаг 1: Откройте настройки проекта
1. Зайдите в [Railway Dashboard](https://railway.app/)
2. Выберите ваш проект с ботом
3. Перейдите во вкладку **"Settings"**

### Шаг 2: Создайте Volume
1. Нажмите **"New Volume"**
2. Настройте параметры:
   - **Name**: `whale-data`
   - **Mount Path**: `/app/data`
   - **Size**: `1GB` (достаточно для users.json)

### Шаг 3: Примените изменения
1. Нажмите **"Create"**
2. Railway автоматически перезапустит приложение
3. Volume будет примонтирован в `/app/data`

## 🔧 Проверка работы Volume

### 1. Проверьте создание директории
В коде бота директория `data` создается автоматически:
```python
# bot.py, функция post_init
Path("data").mkdir(exist_ok=True)
```

### 2. Проверьте запись файлов
После первого запуска бота отправьте `/start` - должен создаться `users.json`

### 3. Проверьте сохранение
Перезапустите приложение в Railway - файл `users.json` должен остаться

## 🚨 Возможные проблемы и решения

### Проблема: "Permission denied"
**Решение:** Убедитесь что права доступа правильные в Dockerfile:
```dockerfile
RUN mkdir -p data logs
```

### Проблема: "Volume not found"
**Решение:** Проверьте что mount path точно `/app/data`

### Проблема: "Data still lost"
**Решение:** Убедитесь что UserStorage пишет в правильный файл:
```python
# monitors/whale_monitor.py
self.filepath = "data/users.json"  # Должен быть в /app/data
```

## 🔄 Backup данных

Для дополнительной надежности:

### 1. Экспорт данных
Добавьте команду `/backup` для экспорта users.json:
```python
async def backup_command(update: Update, context: ContextTypes.DEFAULT_TYPE):
    with open("data/users.json", "r") as f:
        data = f.read()
    await update.message.reply_document(
        document=data,
        filename="users_backup.json"
    )
```

### 2. Регулярный backup
Можно настроить ежедневный backup в Railway через cron.
