# 🔐 Секретные переменные для Railway

## 🎯 Важность правильной настройки

Railway - это продакшен-среда. Ошибки в переменных окружения могут привести к:
- Бот не запустится
- API будет недоступен  
- Данные пользователей потеряются
- Демо провалится

---

## 📋 Обязательные переменные

### 1. BOT_TOKEN (КРИТИЧНО)
**Что:** Токен вашего Telegram бота
**Где взять:** @BotFather → /newbot
**Формат:** `123456789:ABCdefGHIjklMNOpqrsTUVwxyz`

**Проверка:**
```bash
# Проверьте токен локально перед деплоем
curl -X GET "https://api.telegram.org/bot123456789:ABCdefGHIjklMNOpqrsTUVwxyz/getMe"
```

### 2. PORT (Автоматически)
**Что:** Порт для HTTP сервера
**Значение:** Railway устанавливает автоматически
**Код:** `int(os.environ.get("PORT", 8080))`

---

## 🔧 Опциональные переменные

### DEMO_MODE
**Что:** Включить демо-режим
**Значение:** `true`
**Нужно для:** Показа клиенту без реальных API

### WHALE_ALERT_API_KEY
**Что:** Ключ Whale Alert API
**Значение:** Ваш ключ или `demo_key`
**Нужно для:** Реальных данных о китах

### PYTHONUNBUFFERED
**Что:** Немедленный вывод логов
**Значение:** `1`
**Нужно для:** Отладки на Railway

---

## 🛠️ Настройка в Railway Dashboard

### Шаг 1: Variables вкладка
1. Откройте проект на Railway
2. Перейдите во вкладку **"Variables"**
3. Нажмите **"New Variable"**

### Шаг 2: Добавьте переменные

| Variable | Value | Environment |
|----------|-------|-------------|
| `BOT_TOKEN` | `123456789:ABC...` | Production |
| `DEMO_MODE` | `true` | Production |
| `PYTHONUNBUFFERED` | `1` | Production |
| `WHALE_ALERT_API_KEY` | `demo_key` | Production |

### Шаг 3: Проверка
Railway автоматически перезапустит приложение после добавления переменных.

## 🚨 Частые проблемы

### Проблема: "BOT_TOKEN not set"
**Причина:** Переменная не добавлена в Railway
**Решение:** Добавьте `BOT_TOKEN` в Variables

### Проблема: "Invalid bot token"
**Причина:** Неправильный токен
**Решение:** Проверьте токен через curl API

### Проблема: "Application not responding"
**Причина:** Ошибка в коде или переменных
**Решение:** Проверьте логи в Railway Dashboard

### Проблема: "Port already in use"
**Причина:** Конфликт портов
**Решение:** Используйте `os.environ.get("PORT", 8080)`

## 🔐 Безопасность

### Никогда не храните в коде:
```python
# ❌ ПЛОХО
BOT_TOKEN = "123456789:ABC..."

# ✅ ХОРОШО
BOT_TOKEN = os.getenv("BOT_TOKEN")
```

### Защита переменных:
- Railway шифрует переменные
- Доступ только через Dashboard
- Не попадают в Git историю
