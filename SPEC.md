# Whale & Smart Alert Tracker — Спецификация

## 1. Концепция и Видение

**Whale & Smart Alert Tracker** — это мощный Telegram-бот для real-time мониторинга блокчейн-активности и рыночных аномалий. Бот объединяет элегантный дизайн уведомлений с молниеносной скоростью доставки, создавая ощущение "личного радара" для крипто-активностей.

**Ключевое ощущение**: Пользователь чувствует себя "внутри" рынка — каждое крупное движение становится видимым мгновенно, с красивой визуализацией и детальным контекстом.

## 2. Дизайн-система

### Цветовая палитра
- **Primary**: `#1E88E5` (синий — доверие, технологичность)
- **Secondary**: `#00C853` (зелёный — позитивные изменения, рост)
- **Accent**: `#FF6D00` (оранжевый — важные оповещения, внимание)
- **Alert**: `#FF1744` (красный — критические события, падение)
- **Background**: `#0D1117` (тёмный фон — погружение в данные)
- **Surface**: `#161B22` (карточки, блоки информации)
- **Text Primary**: `#E6EDF3` (основной текст)
- **Text Secondary**: `#8B949E` (вторичный текст, метки)

### Типографика
- **Заголовки**: `Montserrat Bold` — 18px
- **Основной текст**: `Inter` — 14px
- **Моноширинный (данные)**: `JetBrains Mono` — 13px
- **Метки**: `Inter Medium` — 12px

### Визуальные элементы
- Эмодзи для быстрой визуальной навигации: 🐋 💰 📊 🔔 📈 📉 ⚡ 🎯
- Градиенты для акцентов (blue → cyan)
- Пунктирные разделители между секциями
- Компактные карточки с тенью для группировки данных

### Анимации
- Мгновенная доставка уведомлений (< 1 сек)
- Плавное появление карточек (200ms ease-out)
- Пульсация для критических алертов

## 4. Функциональность

### Основные команды бота

| Команда | Описание |
|---------|----------|
| `/start` | Приветствие + быстрый старт |
| `/help` | Справка по функциям |
| `/alerts` | Управление подписками |
| `/whales` | Настройка порогов китов |
| `/price` | Настройка price alerts |
| `/status` | Текущий статус мониторинга |
| `/settings` | Общие настройки |

### Типы отслеживаемых событий

#### 1. Whale Transfers (Киты)
- Минимальная сумма: настраивается ($10K — $1M+)
- Поддерживаемые токены: USDT, USDC, ETH, BTC, WBTC, BNB
- Поддерживаемые сети: Ethereum, Tron (USDT), BSC, Polygon
- Фильтры: вход/выход на биржи, межсетевые мосты

#### 2. Price Alerts (Ценовые оповещения)
- Пороги: процент изменения, абсолютная цена
- Пары: любые пары с Binance/KuCoin
- Таймфреймы: 1m, 5m, 15m, 1h

#### 3. Token Unlocks (Разлоки токенов)
- Мониторинг: через Twitter/официальные источники
- Категории: вестинг, team unlock, инвесторы
-threshold: настраиваемая сумма ($100K+)

#### 4. Exchange Flows (Потоки на биржи)
- Отслеживание: Binance, Coinbase, Kraken, Bybit
- Метрики: приток/отток USDT, крупные ордера

### Настройки пользователя
- Выбор сетей для мониторинга
- Настройка порогов для каждого типа
- Выбор формата уведомлений (компактный/детальный)
- Время тишины (no-alert periods)

## 5. Архитектура системы

### Компоненты

```
┌─────────────────────────────────────────────────┐
│                  Telegram Bot                    │
│              (python-telegram-bot)              │
├─────────────────────────────────────────────────┤
│                   Core Engine                    │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐        │
│  │ Whale    │ │ Price    │ │ Unlock   │        │
│  │ Monitor  │ │ Monitor  │ │ Monitor  │        │
│  └────┬─────┘ └────┬─────┘ └────┬─────┘        │
│       │            │            │               │
│  ┌────┴────────────┴────────────┴─────┐        │
│  │         Alert Formatter            │        │
│  └────────────────┬──────────────────┘        │
│                   │                            │
│  ┌────────────────┴──────────────────┐        │
│  │        User Preferences           │        │
│  └──────────────────────────────────┘        │
└─────────────────────────────────────────────┘
```

### Внешние API
- **Whale Alert API**: `https://whale-alert.io` — транзакции китов
- **Binance API**: `https://api.binance.com` — цены, объёмы
- **CoinGecko API**: `https://api.coingecko.com` — дополнительные данные
- **Blockchain Explorers**: Etherscan, Tronscan — валидация транзакций

### Хранение данных
- **User preferences**: JSON-файлы (для MVP), SQLite (масштабирование)
- **Alert history**: Кэш в памяти + файл
- **Session state**: В оперативной памяти

## 6. Технический стек

- **Язык**: Python 3.10+
- **Фреймворк**: python-telegram-bot (async)
- **HTTP клиент**: aiohttp
- **Парсинг**: BeautifulSoup4
- **Логирование**: logging + pretty-telegram-logger
- **Планировщик**: asyncio + apscheduler
